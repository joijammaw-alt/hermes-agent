## 🐞 2. ข้อผิดพลาดที่พบและการแก้ไข (Debug & Optimization)

| ข้อผิดพลาด (Error) | สาเหตุหลัก | วิธีการแก้ไขสำเร็จ |
| :--- | :--- | :--- |
| **`UnicodeEncodeError (cp874)`** | ระบบ Windows ภาษาไทยไม่รองรับการพิมพ์สัญลักษณ์พิเศษ/Emoji (เช่น 🚀, ⚠️, 🎉) ในหน้า Terminal | เพิ่มคำสั่ง Reconfigure เข้ารหัส `sys.stdout/stderr` เป็น **`utf-8`** ตั้งแต่บรรทัดแรกสุดของ [main.py](file:///d:/hermes-agent/main.py) |
| **`Astro Unstyled CSS Bug`** | Astro นำแท็ก `<script>` ของ Tailwind CSS CDN และ Tailwind Config ไปบีบอัดใหม่ ทำให้ระบบสไตล์บนหน้าเว็บล้มเหลว | เพิ่ม Attribute **`is:inline`** เข้าไปในแท็ก `<script>` ในส่วนของ `<head>` ทำให้หน้าเว็บแสดงสไตล์และสีสันได้อย่างสวยงามทันที |
| **`Gemini API Rate Limit (429)`** | คีย์ Gemini Free Tier จำกัดความเร็วไว้ต่ำมากที่ **15 RPM** (15 ครั้งต่อนาที) ทำให้โดนระบบบล็อกเพราะ CrewAI คุยค่อนข้างถี่ | สลับสมองกลหลักของ Agent ไปใช้ **DeepSeek (deepseek-chat)** ด้วยคีย์จริงของท่าน ทำให้ประมวลผลได้อย่างรวดเร็วและไม่ติดลิมิต |

## 🚀 3. วิธีการสั่งใช้งานและรันระบบบนเครื่องของคุณ

### 1) การรันระบบเว็บพัฒนา Astro (ดูหน้าเว็บจริงโฉมใหม่)
เปิด Terminal ในโฟลเดอร์โปรเจกต์ `d:\hermes-agent` แล้วรันคำสั่งจำลองหน้าเว็บจริง:
```bash
npm run dev
```
👉 จากนั้นให้เปิดชมหน้าแรกที่สวยตระการตาได้ที่: **`http://localhost:4321/`**

### 2) การสั่งให้ Hermes Agent รันทำงาน 3 ขั้นตอนใหม่ทั้งหมด
หากมีข้อมูลใหม่หรืออยากให้ Agent คิดรันวิเคราะห์รอบใหม่ (ตรวจสอบให้แน่ใจว่าคีย์ DeepSeek ในไฟล์ `.env` ถูกต้องและมีโควต้าพร้อมใช้) ให้รันคำสั่งนี้ครับ:
```bash
venv\Scripts\python.exe main.py
```

### 3) การทดสอบส่งข่าวสารเดี่ยว ๆ ขึ้น WordPress แบบสแตนด์อโลน (ไม่คุยกับ AI)
หากต้องการส่งข่าวเด่นขึ้น WordPress ตรง ๆ โดยไม่ต้องให้ AI รันประมวลผลใหม่ ให้พิมพ์คำสั่งนี้:
```bash
venv\Scripts\python.exe post_wp.py
```
👉 ลิงก์เข้าชมหน้าเว็บบล็อกหลักของท่าน: **`http://localhost:10004/`**

## 📝 4. ขั้นตอนการพัฒนาระบบโดยละเอียด (Development Journey & Step-by-Step)
เพื่อเป็นประโยชน์ในการศึกษาและปรับปรุงโค้ดในอนาคต นี่คือลำดับขั้นตอนการวิเคราะห์ แก้ไข และพัฒนาที่เกิดขึ้นทั้งหมดในโปรเจกต์นี้ครับ:

### 🔍 ขั้นตอนที่ 1: ตรวจหาบั๊กการแสดงผลแรกสุด (Unicode Debugging)
* **ปัญหาที่พบ**: เมื่อสั่งรัน `python main.py` ครั้งแรก โปรแกรมเกิดการแครชและหยุดทำงานทันทีฟ้องข้อผิดพลาด `UnicodeEncodeError: 'charmap' codec can't encode character '\U0001f680'`
* **การแก้ไข**: เข้าใจระบบ Windows ภาษาไทยว่าใช้โค้ดเพจ `cp874` ซึ่งไม่รองรับอิโมจิจึงได้เขียนชุดรหัสควบคุมเพิ่มเข้าที่บรรทัดบนสุดของ [main.py](file:///d:/hermes-agent/main.py) เพื่อบังคับ Reconfigure `sys.stdout` และ `sys.stderr` ให้เข้ารหัสเป็น **`utf-8`** เสมอ ทำให้รันโปรแกรมผ่านด่านแรกได้สำเร็จ

### 🌐 ขั้นตอนที่ 2: เริ่มต้นติดตั้งโครงการจำลองเว็บ Astro (Initialize Web Environment)
* **ปัญหาที่พบ**: เมื่อสั่งรันเว็บหน้าแรก Astro ด้วยคำสั่งพัฒนาจำลอง `npm run dev` พบปัญหาระบบฟ้องหาไฟล์ `package.json` ไม่พบ (`ENOENT: no such file or directory`) เนื่องจากโฟลเดอร์มีเพียงไฟล์เนื้อหาหน้าแรกของ Astro แต่ยังไม่เคยถูกเซ็ตอัปเป็นโปรเจกต์ Node.js จริง ๆ
* **การแก้ไข**: 
  1. ดำเนินการสร้างไฟล์ [package.json](file:///d:/hermes-agent/package.json) และไฟล์ [astro.config.mjs](file:///d:/hermes-agent/astro.config.mjs) ปลายทางขึ้นมาใหม่ในไดเรกทอรี
  2. รันคำสั่ง `npm install` ในสภาพแวดล้อมจำลองเพื่อดาวน์โหลดและลงทะเบียนชุดประมวลผลของ Astro (330 แพ็กเกจ) ลงเครื่องของท่าน ทำให้การสั่งรันเว็บด้วยคำสั่ง `npm run dev` ทำงานได้จริงทันที

### 🧠 ขั้นตอนที่ 3: สลับและยกระดับสมอง AI ไปยัง DeepSeek Chat (LLM Migration)
* **ปัญหาที่พบ**: เนื่องจากคีย์ของ Google Gemini ที่มีอยู่เป็นคีย์ระดับฟรี (Free Tier) ซึ่งมีการจำกัดโควต้าเรียกใช้อย่างแน่นหนาที่ **15 RPM (15 ครั้งต่อนาที)** และโควต้าวันเต็ม ส่งผลให้ระบบ CrewAI ที่จำเป็นต้องส่งคำถามทีละมาก ๆ ในขั้นตอนประเมินแผนงาน เกิดแครชด้วยรหัส 429
* **การแก้ไข**: 
  1. ดำเนินการนำ API Key ของ DeepSeek (`sk-...`) ตัวจริงของท่านไปเพิ่มไว้ในไฟล์การตั้งค่าระบบ [.env](file:///d:/hermes-agent/.env)
  2. เปลี่ยนการเชื่อมต่อ LLM ภายในไฟล์ [main.py](file:///d:/hermes-agent/main.py) จาก Google Gemini เดิมไปชี้ช่องทางเรียกใช้ **`deepseek/deepseek-chat` (DeepSeek V4-Flash)** โมเดลประสิทธิภาพสูงของ DeepSeek แทน ซึ่งรองรับการรันงานได้อย่างรวดเร็ว แม่นยำ และแทบไม่มีการจำกัด Rate Limit กวนใจ

### 🎨 ขั้นตอนที่ 4: แก้ไขบั๊กการเรนเดอร์สไตล์ และอัปเดตฟีเจอร์พรีเมียม (Tailwind & Client Side Filtering)
* **ปัญหาที่พบ**: เมื่อเปิดดูที่อยู่เว็บจำลอง `localhost:4321` ครั้งแรก พบว่าหน้าจอแสดงผลเพียงข้อความพื้นฐานสีดำ ไม่มีกริด สีสัน หรือธีมใด ๆ ทำงาน
* **การแก้ไข**:
  1. วิเคราะห์ระบบของเฟรมเวิร์ก Astro พบว่าระบบคอมไพเลอร์ของ Astro พยายามรวบและบีบอัดแท็ก `<script>` ของ Tailwind CSS ส่งผลให้ระบบสไตล์ของเบราว์เซอร์ไม่ถูกเตรียมล่วงหน้า
  2. ดำเนินการเติมคุณสมบัติ **`is:inline`** เข้าไปในแท็กสคริปต์ของ Tailwind CDN และตัวคอนฟิกสไตล์ในหน้าหัวของไฟล์ [index.astro](file:///d:/hermes-agent/src/pages/index.astro) ส่งผลให้เบราว์เซอร์เปิดระบบสไตล์ของ Tailwind ขึ้นมาเรนเดอร์ธีมมืดและสีสันนีออนได้ทันทีที่โหลดหน้าเว็บ
  3. สั่งให้ตัว Agent (สมองใหม่ DeepSeek) ประมวลผลขั้นสูงเพื่อเพิ่มลูกเล่นปุ่มเลือกตัวกรองกลุ่มข่าวประชาสัมพันธ์ (Dynamic Filter Buttons) หน้าแรกเว็บให้สามารถคลิกเลือกตามหมวดหมู่ได้ทันใจแบบไม่ต้องรอโหลดหน้าใหม่

### 🚀 ขั้นตอนที่ 5: เชื่อมต่อเว็บบล็อก และส่งมอบงานจริง (Integration & Final Verification)
* **การทำงาน**: สตาร์ทเปิดระบบ Local WordPress พอร์ต `10004` ผ่านโปรแกรม Local by Flywheel และสั่งรันระบบท่อส่งงานทั้งหมด
* **ผลสำเร็จ**: 
  * ระบบ Hermes ทำการดึงและขูดบันทึกข้อมูลหน้าข่าวสารโรงเรียนวัชระแยกเก็บเป็นเอกสารคุณภาพสูงใน Obsidian ได้ถึง 6 ไฟล์ 
  * เรียบเรียงและส่งขึ้นเผยแพร่บทความบนเว็บบล็อกจริงได้สำเร็จลุล่วงครบถ้วนถึง 3 ข่าวเด่นหลัก (Post ID 7, 8, 9)
  * หน้าเว็บไซต์ Astro แสดงข้อมูลข่าวสารเชื่อมโยงแอนิเมชัน และปุ่มตัวกรองทำงานได้อย่างราบรื่น ไร้ข้อผิดพลาด
